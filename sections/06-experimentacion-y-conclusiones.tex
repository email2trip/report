\section{Resultados Experimentales}%
\label{sec:experimentation-and-results}

Para el despliegue del \textit{pipeline} de extracción y recomendaciones se decidió implementar una aplicación web
donde un usuario puede registrarse y luego acceder con un usuario y contraseña.
Ambas pantallas se pueden ver en la \autoref{fig:web-register-login}.

    \end{minipage}
    \hfill
    \begin{minipage}[t]{0.49\textwidth}

    \end{minipage}
    \figcaption{Pantallas de registro y login}%

Luego al presionar ``New Trip'' podrá arrastrar al navegador un archivo \texttt{.eml} con el correo electrónico sobre el
cual se desea hacer la extracción de información (Ver \autoref{fig:web-upload}).

\begin{figure}[H]
    \centering
    \includegraphics[width=1.00\textwidth]{images/web-upload}
    \figcaption{Página para arrastrar un archivo \texttt{.eml} de vuelos o trenes.}%
    \label{fig:web-upload}
\end{figure}

El usuario deberá exportar el correo en formato \texttt{.eml} usando la opción exportar de un cliente de escritorio de
correo electrónico (como Microsoft Outlook, Mozilla Thunderbird o Apple Mail) o desde un webmail (como GMail u
Outlook.com).
Elegido el archivo basta con presionar ``Create Trip'' para que se dispare el \textit{pipeline} de extracción.

Todos los correos electrónicos procesados con éxito son asociados al usuario logueado y persistidos en la base de datos
para que puedan ser consultados en la sección ``My Trips'' que puede apreciarse en la \autoref{fig:web-home}.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=1.00\textwidth]{images/web-home}
    \figcaption{Página Principal}%
    \label{fig:web-home}
\end{figure}

Allí se listan todos los viajes creados a partir de la extracción, donde para cada correo electrónico se genera un
viaje distinto.
Además se ofrece el filtro por año de viaje.
Para cada viaje se indica el destino final (la ciudad correspondiente al aeropuerto de destino del último segmento),
el país del destino final, un nombre autogenerado (por ejemplo ``Trip to Montréal'') que el usuario puede modificar,
las fechas de inicio y fin del viaje, la cantidad de segmentos que tiene el viaje y la cantidad de itinerarios de POIs
generados.

En la página principal, además de la sección ``My Trips'', se cuenta con la sección ``My Interests''.
Tal como se puede ver en la \autoref{fig:web-interests}, el usuario puede consultar sus dos intereses seleccionados y
cambiarlos por cualquiera de las cinco opciones disponibles.
De acuerdo a lo ya mencionado en~\autoref{par:poi}, estas preferencias son utilizadas para la obtención de
recomendaciones de POIs.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=1.00\textwidth]{images/web-interests}
    \figcaption{Edición de los intereses del usuario}%
    \label{fig:web-interests}
\end{figure}

Al hacer click en uno de los viajes de ``My Trips'' se ofrece la vista con el detalle de un viaje.
En la parte superior se ofrecen opciones para renombrar el viaje y para eliminarlo.
En la \autoref{fig:web-trip-flights} se puede apreciar la sección ``Flights'', que cuenta con un mapa cada uno de los
segmentos de vuelo del viaje, donde para cada segmento, se lista toda la información relevante que extrajo el
\textit{pipeline}.
Para el mapa se utilizó Leaflet con un mapa interactivo que utiliza una cartografía base minimalista donde sólo aparecen
límites y nombres de países, ciudades y provincias importantes.
Al final se ofrece también el nombre del pasajero y el código de reserva.

\begin{figure}[htbp]
    \centering
    \includegraphics[width=1.00\textwidth]{images/web-trip-flights}
    \figcaption{Detalles de los segmentos aéreos del viaje}%
    \label{fig:web-trip-flights}
\end{figure}

En caso de mostrar una reserva de trenes de la red canadiense se dibujan en el mapa las líneas correspondientes a las
vías.
Esto es posible dado que el archivo GTFS mencionado en \autoref{subpar:extraction-trains} incluye un archivo
\texttt{shapes.txt} con las figuras geométricas necesarias para dibujarlas en el mapa.
Parte del trazado de la red de trenes de la costa este de Canadá se puede ver en la \autoref{fig:web-trip-trains} donde
se incluye un tramo que conecta la ciudad de Montreal con la capital del país (Ottawa).

\begin{figure}[H]
    \centering
    \includegraphics[width=1.00\textwidth]{images/web-trip-trains}
    \figcaption{Detalles de los segmentos de tren}%
    \label{fig:web-trip-trains}
\end{figure}

En la \autoref{fig:web-trip-activities} se puede apreciar la sección ``Activities'', que es la que dispara el sistema de
recomendaciones de POIs usando un LLM\@.
En primer lugar se ofrece la opción de elegir la cantidad de días deseada (entre 1 y 7) y luego corresponde elegir la
ciudad sobre la cual se buscan recomendaciones.
Sólo se ofrecen como opciones de ciudades a aquellas que corresponden con el o los aeropuertos de destino.
Una vez definidos los días y la ciudad basta con presionar la opción ``Generate'' para conseguir un itinerario con las
recomendaciones basadas en los intereses del usuario indicados en ``My Interests''.

Las recomendaciones se organizan por días (se puede seleccionar ``Day 1'', ``Day 2'', etc.) donde cada día cuenta con un
título y subtítulo que actúan como una breve descripción de las actividades del día.
Se ofrece un mapa interactivo donde se ubican geográficamente los puntos de interés de forma que el usuario puede
utilizar las flechas para ir conociendo el detalle de cada uno de los puntos de interés recomendados para ese día.
El detalle del POI consiste en el nombre, si está asociado o no a uno de los intereses de preferencia del usuario y una
breve descripción.
Para este mapa también se utiliza Leaflet, pero ahora con un mapa base de OpenStreetMap de tipo ciudad donde se muestra
por ejemplo los nombres de barrios, calles, monumentos y demás puntos de interés.

Si el usuario lo desea puede presionar ``Save'' y así persistir el itinerario completo con las recomendaciones obtenidas
para consultarlo más adelante.
La implementación permite almacenar un itinerario de recomendaciones por ciudad destino por viaje.
